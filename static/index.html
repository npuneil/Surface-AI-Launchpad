You have an NPU!

Your device is a Copilot+ PC with dedicated AI silicon

Your Neural Processing Unit (NPU) is a dedicated chip designed to run AI workloads with exceptional power efficiency. Unlike running AI on your CPU (which heats up and drains battery) or GPU (which competes with display tasks), the NPU handles AI inference while barely sipping power. This means always-on AI features like Recall, Click to Do, and Semantic Search can run continuously in the background without impacting your battery life or system performance.

NPU models in Foundry Local use the QNN (Qualcomm Neural Network) execution provider, specifically optimized for the Hexagon processor in your Snapdragon chip.
💡

No NPU Detected

You can still run on-device AI using your CPU

While you don't have an NPU, Foundry Local has an excellent selection of CPU-optimized models. Your CPU handles a wider range of models (including larger ones) — the trade-off is higher power consumption and slightly slower inference compared to NPU-accelerated models. Check the Model Recommendations page for the best picks for your hardware.

Foundry Local

Microsoft's local model runtime — run SLMs entirely on your device

Run AI models locally, no cloud required

Foundry Local is an OpenAI-compatible local inference server. It manages model downloads, hardware optimization (CPU/NPU), and exposes a standard API you can use from any language.

📦

Installation

Get Foundry Local running on your machine
1

Install via winget

Open a terminal and run:

winget install Microsoft.FoundryLocal
2

Start the service

Foundry Local auto-starts when needed, but you can manually control it:

foundry service start
3

List available models

See all models optimized for your hardware (CPU + NPU if available):

foundry model list
4

Run a model interactively

Start chatting with a model directly in your terminal:

foundry model run phi-4-mini
For NPU models, add --device NPU: foundry model run qwen2.5-7b --device NPU
🔌

Using the API

OpenAI-compatible REST API for your apps

Foundry Local exposes an OpenAI-compatible API. Use it with the OpenAI Python SDK, curl, or any HTTP client. The service URL has a dynamic port — discover it with foundry service status.

Py

Python with OpenAI SDK

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:PORT/v1",
    api_key="not-needed"
)

response = client.chat.completions.create(
    model="Phi-4-mini-instruct-generic-cpu:5",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
C#

C# with Microsoft.Extensions.AI

using Microsoft.Extensions.AI;

var client = new FoundryLocalClient("phi-4-mini");
var chatClient = client.CreateChatCompletionClient();

var response = await chatClient.GetResponseAsync(
    "What can you help me with?");
Console.WriteLine(response);

AI Toolkit for VS Code

Fine-tune, test, and deploy AI models right from your editor

Your AI development workbench

AI Toolkit is a VS Code extension that brings model exploration, prompt testing, fine-tuning, and deployment into a single, integrated experience.

🚀

Getting Started

Set up AI Toolkit in minutes
1

Install the Extension

Open VS Code and install AI Toolkit from the Extensions marketplace, or run:

code --install-extension ms-windows-ai-studio.windows-ai-studio
2

Browse the Model Catalog

Click the AI Toolkit icon in the sidebar. Browse models from Hugging Face, ONNX model zoo, and Azure AI — filtered by what runs on your hardware.

3

Test in the Playground

Use the built-in Model Playground to send prompts and compare model outputs. Adjust temperature, max tokens, and system prompts interactively.

4

Fine-tune with LoRA

Fine-tune models on your own data using QLoRA — memory-efficient fine-tuning that works even on devices with limited VRAM. AI Toolkit handles the training setup.

Recall

AI-powered photographic memory for your PC — powered by your NPU

🔄 Find anything you've ever seen on your PC

Recall takes periodic snapshots of your screen and uses on-device AI to make everything searchable by natural language. Ask "that email about the project deadline" and Recall finds it.

🔒

Privacy First

Everything stays on your device

Recall processes everything locally on your NPU. Screenshots never leave your device, are encrypted with Windows Hello, and stored in a protected container. You have full control over what gets saved and can delete snapshots at any time.

Privacy FeatureDetail
🔐 EncryptionSnapshots encrypted with Windows Hello (biometric/PIN) — even admins can't access your data
🖥️ On-device AIOCR and semantic indexing runs entirely on the NPU — no cloud processing
🚫 Sensitive contentAutomatically filters passwords, credit cards, and private browsing sessions
🗑️ Full controlPause anytime, exclude apps/websites, delete specific snapshots or entire history
⚙️

How to Enable Recall

Get started on your Copilot+ PC
1

Check requirements

You need a Copilot+ PC with an NPU (40+ TOPS), 16GB+ RAM, 256GB+ storage, and BitLocker/Device Encryption enabled.

2

Open Settings

Go to Settings → Privacy & Security → Recall & snapshots

3

Enable and configure

Toggle Recall on and set your preferences: storage allocation (25-150GB), excluded apps, and excluded websites. Authenticate with Windows Hello.

4

Search your timeline

Press Win + J to open Recall, or click the Recall icon in the taskbar. Type a natural language query like "presentation about Q3 results" to find it.

🧠

How It Works Under the Hood

The NPU does the heavy lifting
📸

Screen Capture

Periodic screenshots are taken (every ~5 seconds when content changes). Images are immediately encrypted before any processing.

📝

NPU-Powered OCR

The NPU runs optical character recognition on each snapshot, extracting all visible text with minimal power draw. This enables text search across screenshots.

🔗

Semantic Embedding

Extracted text is converted into semantic vectors (embeddings) by the NPU, enabling meaning-based search — so "vacation photos" finds screenshots of beach pictures even if those words never appeared on screen.

Click to Do

Context-aware AI actions on anything visible on your screen

👆 Point at anything, get smart actions

Click to Do analyzes on-screen content in real-time using your NPU and suggests intelligent actions — summarize text, edit images, search the web, and more.

⚡

How to Use Click to Do

Activate and interact
1

Activate the overlay

Press Win + Mouse Click or Win + Q to activate Click to Do. A translucent overlay appears on top of your current screen.

2

Select content

Click on any text, image, or UI element. Click to Do uses the NPU to analyze what you're pointing at — reading text, identifying images, and understanding context.

3

Choose an action

A context menu appears with smart suggestions based on the content type. Actions vary depending on whether you selected text, an image, or a link.

🎯

Available Actions

What Click to Do can do with different content
Content TypeActions
📝 TextSummarize, rewrite, explain, translate, copy, search the web, send to Copilot
🖼️ ImagesRemove background, erase objects, search for visually similar images, save, edit in Photos
🔗 URLs / LinksOpen, copy, search related content
📊 Data / TablesCopy as structured data, summarize, analyze trends
📧 Contact InfoCreate contact, send email, call, add to calendar
Click to Do works across any application — it operates on the screen pixels, not app APIs. This means it works with PDFs, images, legacy desktop apps, and even remote desktop sessions.
⚙️

Configuration

Customize Click to Do behavior

Navigate to Settings → Bluetooth & devices → Touchpad → Click to Do (or search "Click to Do" in Settings). You can configure the activation gesture, choose default actions for different content types, and exclude specific applications.

Semantic Search

Search by meaning, not just keywords — powered by on-device AI

🔎 "Find that document about budget planning"

Windows Semantic Search understands what you mean, not just what you type. It uses the NPU to generate embeddings for your files, so you can search by concept instead of exact filenames.

🧠

How Semantic Search Works

AI-enhanced Windows Search
1

Content Indexing

Windows scans your documents, emails, and files. The NPU generates semantic embeddings — numerical vectors that capture meaning — for each piece of content. This happens in the background with negligible performance impact thanks to NPU efficiency.

2

Natural Language Queries

When you search, your query is also converted to an embedding. The system finds documents whose meaning is closest to your query — even if they don't contain the exact words you typed.

3

Ranked Results

Results are ranked by semantic similarity, combining traditional keyword matching with AI understanding. This means a search for "vacation plans" can find a file named "summer-trip-itinerary.docx" even though those words don't overlap.

💡

Semantic Search is integrated into the Windows Start menu search, File Explorer, and Settings. Just start typing naturally — the AI enhancement works automatically on Copilot+ PCs.
🔧

For Developers

Build semantic search into your apps

You can leverage on-device embeddings for your own applications using Windows Copilot Runtime APIs. Generate text embeddings locally using the NPU for vector search, RAG pipelines, and intelligent document retrieval — all without cloud costs or latency.

The Windows Copilot Runtime TextEmbedding API generates 384-dimensional vectors using the NPU-accelerated model. Use it with any vector database (FAISS, ChromaDB, Qdrant) for local RAG applications.
